# 信息安全与保密管理规定

> 本规定旨在规范星河科技有限公司全体员工的信息安全行为，保护公司核心数据资产、客户隐私和商业机密，防止数据泄露、滥用和损坏。本规定适用于所有正式员工、外包人员、实习生及访客。

## 一、信息资产分级

### 1.1 数据分级标准

公司所有信息资产按敏感程度分为四级：

| 等级 | 名称 | 示例 | 处理要求 |
|---|---|---|---|
| L1 | 公开信息 | 官网介绍、新闻稿 | 无限制 |
| L2 | 内部信息 | 组织架构、内部通知 | 仅公司内部 |
| L3 | 机密信息 | 财务数据、客户名单、薪资 | 部门内授权访问 |
| L4 | 绝密信息 | 核心源代码、并购计划、密钥 | 特定人员授权 |

### 1.2 资产归属

所有在公司设备、公司账号、公司云盘中产生和处理的信息资产均归公司所有。员工离职时须完整移交，不得擅自删除、拷贝或销毁。

## 二、账号与密码管理

### 2.1 密码强度要求

- 长度 ≥ 12 位
- 必须包含大写字母、小写字母、数字、特殊字符中的 3 种以上
- 不得使用生日、姓名拼音、连续数字等弱口令
- 不得在多个系统复用同一密码

### 2.2 密码更换周期

| 账号类型 | 更换周期 |
|---|---|
| 普通业务系统 | 90 天 |
| VPN / 跳板机 | 60 天 |
| 数据库 / 生产服务器 | 30 天 |
| 管理员账号 | 30 天 |

### 2.3 多因素认证（MFA）

以下场景必须启用 MFA：
- 公司邮箱
- VPN 远程接入
- 云服务控制台（阿里云、AWS）
- 代码仓库（GitLab、GitHub）
- 生产环境服务器

### 2.4 账号共享禁令

严禁将个人账号借给他人使用，严禁使用他人账号登录任何系统。账号的所有操作由账号持有人负责。

## 三、设备使用安全

### 3.1 办公电脑

- 必须安装公司统一部署的杀毒软件（默认 Microsoft Defender 或卡巴斯基）
- 必须开启全盘加密（macOS FileVault / Windows BitLocker）
- 系统补丁需在发布后 30 天内安装
- 严禁安装来源不明的软件、破解工具、外挂程序

### 3.2 移动存储

- U 盘、移动硬盘须先查毒再使用
- L3 及以上数据不得拷贝到个人 U 盘
- 提倡使用公司云盘替代物理介质

### 3.3 BYOD（自带设备）

如需使用个人电脑/手机处理公司业务，须：
1. 在 IT 服务台登记设备信息
2. 安装 MDM（移动设备管理）软件
3. 签署《BYOD 安全协议》
4. 接受 IT 部门远程擦除权限（仅限公司数据区域）

## 四、网络与通信安全

### 4.1 办公网络

- 内部网络与访客 WiFi 严格隔离
- 严禁私自搭建代理、VPN、热点绕过公司防火墙
- 严禁将办公网线接入未授权设备

### 4.2 远程办公

- 必须通过公司 VPN 接入内网
- 严禁在公共 WiFi 下直接处理 L3 及以上数据
- 视频会议不得录屏，会议内容不得对外传播

### 4.3 邮件安全

- 发送 L3 数据须加密附件并单独发送解压密码
- 警惕钓鱼邮件，可疑邮件不要点击链接或下载附件
- 外部邮件带 `[External]` 标识，重要业务邮件须核实发件人

## 五、数据保护

### 5.1 客户数据

- 客户个人信息的收集、存储、使用须符合《个人信息保护法》
- 客户数据脱敏后才可用于测试、培训、数据分析
- 严禁将客户数据带出公司或用于非业务用途

### 5.2 数据备份

- 工作文档优先存公司云盘（自动每日备份）
- 本地重要文件每周手动备份到公司 NAS
- 数据库管理员负责生产数据库的每日全量 + 实时增量备份

### 5.3 数据销毁

- L3 及以上数据删除须走 IT 工单审批
- 物理硬盘报废前须消磁或物理粉碎
- 云端数据删除后须清空回收站并联系云厂商彻底清理

## 六、保密协议与竞业

### 6.1 保密协议

入职时所有员工须签署《保密协议》，约定：
- 任职期间及离职后 2 年内不得泄露公司商业机密
- 违约赔偿金额为年薪的 3-10 倍

### 6.2 竞业限制

涉及核心技术的岗位（P7 及以上技术专家、总监及以上管理岗）须额外签署《竞业限制协议》：
- 离职后 1 年内不得加入直接竞争对手
- 竞业期间公司按月薪 50% 支付补偿金

### 6.3 离职交接

离职员工须：
- 7 个工作日内完成工作交接
- 归还所有公司设备、门禁卡、证书
- 签署《离职保密承诺书》
- IT 部门停用所有系统权限

## 七、违规处理

### 7.1 违规等级

| 等级 | 情形示例 | 处理方式 |
|---|---|---|
| 轻微 | 密码未按时更换、电脑未锁屏 | 警告 + 培训 |
| 一般 | U 盘未查毒、共用账号 | 记过 + 扣绩效 |
| 严重 | 数据外泄、私搭代理 | 解除劳动合同 |
| 极严重 | 出卖商业机密、入侵系统 | 移交司法机关 |

### 7.2 举报与核查

发现违规行为可通过以下渠道举报：
- 直接上级
- IT 安全部门邮箱：security@xinghe.com
- 匿名举报平台（公司 OA → 合规举报）

公司承诺保护举报人，举报查处后给予举报人最高 5000 元奖励。

## 八、应急响应

### 8.1 安全事件上报

发生以下事件须在 1 小时内上报 IT 安全部门：
- 电脑中毒、勒索软件
- 账号被盗、密码泄露
- 设备丢失（含手机、笔记本）
- 怀疑被钓鱼

上报电话：12345（24 小时）

### 8.2 应急流程

1. 隔离：断网、停用账号
2. 取证：保留日志、磁盘镜像
3. 评估：影响范围、数据敏感度
4. 处置：清理威胁、恢复系统
5. 复盘：根因分析、改进措施

## 九、附则

1. 本规定由信息安全委员会负责解释
2. 本规定每年评估修订一次
3. 员工须每年参加 1 次信息安全培训并考核合格
