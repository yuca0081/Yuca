# 知识库模块功能清单（yuca-knowledge）

> 本文档梳理 `yuca-backend/yuca-knowledge` 模块已实现的功能及其相关代码文件，便于快速了解模块能力范围。

## 模块定位

知识库模块负责 **文档 ingest（接入）→ 切片 → 向量化 → 存储 → 检索** 的全链路，是 RAG 系统的数据底座：

- 数据库：PostgreSQL + pgvector（1024 维向量 + HNSW 余弦距离索引）
- 对象存储：MinIO（原始文件归档）
- 模型：DashScope `text-embedding-v3`（嵌入）+ 可选 Cross-Encoder（精排）
- 上游依赖：`yuca-infrastructure`（存储/安全）、`yuca-ai`（Embedding/Rerank 抽象）

---

## 功能一：知识库管理（CRUD）

用户级别的知识库容器，按 `userId` 严格隔离。

| 子功能 | 说明 | 接口 |
|---|---|---|
| 创建知识库 | 同用户下名称唯一约束；返回 kbId | `POST /knowledge-base/create` |
| 更新知识库 | 名称冲突校验（排除自身）、描述更新 | `POST /knowledge-base/{id}/update` |
| 删除知识库 | 逻辑删除（`deleted=1`） | `POST /knowledge-base/{id}/delete` |
| 获取详情 | 含权限校验 | `GET /knowledge-base/{id}` |
| 列表查询 | 按 `updatedAt` 倒序 | `GET /knowledge-base/list` |

**相关代码文件**
- `controller/KnowledgeBaseController.java` — REST 接口入口
- `service/KnowledgeBaseService.java` — 业务逻辑（继承 MyBatis-Plus `ServiceImpl`）
- `entity/KnowledgeBase.java` — 数据库实体（`@TableName("knowledge_base")`、`@TableLogic`）
- `mapper/KnowledgeBaseMapper.java` — MyBatis-Plus Mapper
- `dto/request/CreateKnowledgeBaseRequest.java` — 创建请求（`@NotBlank`、`@Size(max=100)`）
- `dto/request/UpdateKnowledgeBaseRequest.java` — 更新请求
- `dto/response/KnowledgeBaseResponse.java` — 响应 DTO
- DDL: `yuca-app/src/main/resources/table/knowledge/knowledge_base.sql`

## 功能二：文档上传与处理

文档接入入口，承担 "上传 → 解析 → 切片 → 向量化 → 入库" 全流程。

| 子功能 | 说明 | 接口 |
|---|---|---|
| 上传文档 | 单文件 ≤ 50MB；存 MinIO、解析、批量 embedding | `POST /knowledge-doc/upload` |
| 文档分页查询 | 默认 `current=1, size=10`，按 `createdAt` 倒序 | `GET /knowledge-doc/pageList` |
| 批量删除文档 | 逻辑删除文档 + 关联切片级联清理 | `POST /knowledge-doc/delete` |
| 查看切片列表 | 按 `chunkIndex` 升序返回章节树/扁平切片 | `GET /knowledge-doc/{docId}/chunks` |

**相关代码文件**
- `controller/KnowledgeDocController.java` — REST 接口入口
- `service/KnowledgeDocService.java` — 上传/删除/查询主流程
- `entity/KnowledgeDoc.java` — 文档实体（含 `filePath`、`chunkCount`、`dataSource`）
- `entity/KnowledgeChunk.java` — 切片实体（含 `embedding Double[]`、章节树字段）
- `mapper/KnowledgeDocMapper.java` — 文档 Mapper
- `mapper/KnowledgeChunkMapper.java` — 切片 Mapper（含自定义 SQL）
- `resources/mapper/knowledge/KnowledgeChunkMapper.xml` — 向量/关键词检索 SQL
- `dto/response/KnowledgeDocResponse.java` — 文档响应 DTO
- `dto/response/SearchResultResponse.java` — 检索结果响应 DTO
- DDL: `yuca-app/src/main/resources/table/knowledge/knowledge_doc.sql`、`knowledge_chunk.sql`

**文件存储约定**：MinIO 对象路径 `knowledge/{kbId}/{原始文件名}`，依赖 `FileStorageService`（位于 `yuca-infrastructure`）。

## 功能三：文档切片（章节树 + 字符切片）

根据文档格式选择切片策略：**Markdown 走章节树**，其他格式或无标题 Markdown 降级为**字符切片**。

### 3.1 Markdown 章节树切片（主路径）

基于 H1–H6 标题层级的**独占式切片**——每个节点只存自己标题后到下一任意级别标题前的正文（不含子节点正文），避免 embedding 信号重复。

核心算法：
- 逐行扫描 + 代码块围栏（``` / ~~~）状态机过滤误判
- 栈算法建父子关系：新节点弹出栈顶所有 `level ≥ 自己` 的节点
- 面包屑：根节点 = title；非根 = `parent.breadcrumb + " > " + title`
- 前导无标题内容生成虚拟 "简介" 根节点

### 3.2 扁平字符切片（降级路径）

`maxSize=100`、`overlap=10` 的滑窗切片，等价替换 LangChain4j 的 `DocumentByCharacterSplitter`。切片结果包装为 `headingLevel=0` 的 `ChapterNode`，复用同一套存储逻辑。

**相关代码文件**
- `document/MarkdownChapterTreeBuilder.java` — 章节树构造器（栈算法 + 代码块过滤）
- `document/DocumentByCharacterSplitter.java` — 字符切片器（滑窗 + overlap）
- `document/ChapterNode.java` — 章节树内存 DTO（含 `embeddingText()` 用于拼装嵌入源文本）
- `document/Document.java` — 文档包装 record（替换 LangChain4j `Document`）
- `config/KnowledgeConfig.java` — 注册 `MarkdownChapterTreeBuilder` 为 Bean

### 3.3 章节树存储

`KnowledgeDocService.saveChapterNodes` 实现：
1. DFS 前序扁平化所有节点
2. 按 `ChapterNode.embeddingText()` 批量生成嵌入
3. DFS 前序递归插入，回填 id 作为子节点的 `parent_id`

**相关代码文件**
- `service/KnowledgeDocService.java` — `saveChapterNodes`、`flattenDfs`、`insertNodeRecursive`、`countNodesDfs`

## 功能四：向量嵌入生成

调用 DashScope `text-embedding-v3`（1024 维）生成切片向量。

- **批量分片**：单次最多 25 条（DashScope API 上限），超出自动分批（`EMBED_BATCH_SIZE=25`）
- **精度保持**：以 `Double[]` 形式存储，避免浮点精度损失
- **TypeHandler**：通过 `PGVectorTypeHandler` 与 pgvector 类型互转

**相关代码文件**
- `service/KnowledgeDocService.java` — `embedInBatches` 分批调用
- `entity/KnowledgeChunk.java` — `@TableField(typeHandler = PGVectorTypeHandler.class)` 声明
- 上游依赖：`yuca-ai` 模块的 `EmbeddingService`

## 功能五：知识检索（三级流水线 RAG）

`KnowledgeRetrievalService` 实现 `yuca-ai` 的 `RetrievalService` 接口，供 Assistant 模块调用。

```
查询文本
  ↓
[1] 双路召回 (Bi-Encoder, topK=20, 阈值=0.3)
    ├─ 向量召回 (pgvector 余弦距离 + HNSW)
    └─ 关键词召回 (zhparser 中文分词全文搜索)
  ↓
[2] RRF 融合 (k=60)，取前 20 条作为候选池
  ↓
[3] Cross-Encoder 精排 (可选，reranker.enabled=true 时启用)
  ↓
[4] 截取 topN 返回
```

**降级策略**：reranker 未启用或候选 ≤ 1 时自动跳过精排，走纯 RRF 顺序。

**相关代码文件**
- `service/KnowledgeRetrievalService.java` — 检索主流程 + RRF 融合算法
- `mapper/KnowledgeChunkMapper.java` — `searchSimilar` / `searchSimilarGlobal` / `searchByKeyword`
- `resources/mapper/knowledge/KnowledgeChunkMapper.xml` — 三段检索 SQL（pgvector `<=>` 距离、`plainto_tsquery`、`ts_rank_cd`）
- `dto/request/SemanticSearchRequest.java` — 检索请求 DTO（`kbId`、`query`、`topK`、`threshold`）
- 上游依赖：`yuca-ai` 的 `EmbeddingService`、`RerankService`（通过 `ObjectProvider` 可选注入）

## 功能六：权限与数据隔离

- **用户隔离**：所有写操作和查询都基于 `SecurityUtils.getCurrentUserId()` + `userId` 比对
- **越权保护**：`KnowledgeBase.userId` 与当前用户不符时抛 `BusinessException(FORBIDDEN)`
- **逻辑删除**：实体使用 `@TableLogic`，`deleted=1` 表示已删除，查询自动过滤
- **接口保护**：所有 Controller 方法均标注 `@RequireLogin`

**相关代码文件**
- 全部 Controller 类、Service 类
- 依赖：`yuca-common` 的 `@RequireLogin`、`BusinessException`、`ErrorCode`
- 依赖：`yuca-infrastructure` 的 `SecurityUtils`、`JwtAuthenticationFilter`
