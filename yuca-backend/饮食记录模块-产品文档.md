# 饮食记录模块 — 产品需求文档

> 版本：v1.0
> 日期：2026-04-24
> 状态：规划中

---

## 一、模块概述

### 1.1 背景

为用户提供每日饮食记录与热量追踪功能，帮助用户了解自己的饮食结构，控制热量摄入，达成健康目标。模块后期将接入 AI 能力，支持自然语言录入和智能饮食建议。

### 1.2 目标用户

个人用户（复用现有用户体系，支持多用户独立使用）。

### 1.3 设计原则

- 操作简单：手动录入流程尽量简短，减少用户填写成本
- 数据清晰：热量和营养素数据直观展示，一目了然
- 可扩展：为后续 AI 功能预留接口和数据结构

## 二、功能模块

### 模块一：饮食记录

#### 2.1.1 功能说明

用户可以按日期、按餐次记录自己吃了什么食物及食用量，系统自动计算热量和营养素。

#### 2.1.2 记录字段

| 字段 | 说明 | 必填 |
|------|------|------|
| 日期 | 默认当天，可选择历史日期 | 是 |
| 餐次 | 早餐 / 午餐 / 晚餐 / 加餐 | 是 |
| 食物名称 | 手动输入（后续对接食物库） | 是 |
| 食用量 | 数值，单位 g / 份 / ml（默认 g） | 是 |
| 热量 (kcal) | 手动输入或由食物库自动填充 | 是 |
| 蛋白质 (g) | 手动输入或由食物库自动填充 | 否 |
| 脂肪 (g) | 手动输入或由食物库自动填充 | 否 |
| 碳水 (g) | 手动输入或由食物库自动填充 | 否 |
| 备注 | 自由文本，补充说明 | 否 |

#### 2.1.3 操作功能

- **新增记录**：选择日期和餐次，填写食物和营养信息，保存
- **编辑记录**：修改已有记录的任意字段
- **删除记录**：删除单条记录（需二次确认）
- **查看记录列表**：按日期查看当日所有饮食记录，按餐次分组展示
- **日期切换**：支持在记录页面切换日期，查看历史记录

#### 2.1.4 交互细节

- 新增记录时，日期默认当天，餐次根据当前时间自动推荐（6-10点推荐早餐，11-14点推荐午餐，17-20点推荐晚餐，其余推荐加餐）
- 食用量输入时，提供常用快捷选项（如"1碗""1个""1份"）供快速选择
- 热量和营养素支持手动填写，预留食物库自动填充的扩展点

### 模块二：每日统计

#### 2.2.1 功能说明

汇总当日所有饮食记录，展示总摄入量和各餐次分布，让用户快速了解当日饮食情况。

#### 2.2.2 统计指标

| 指标 | 说明 |
|------|------|
| 总热量 | 当日所有记录热量之和（kcal） |
| 总蛋白质 | 当日所有记录蛋白质之和（g） |
| 总脂肪 | 当日所有记录脂肪之和（g） |
| 总碳水 | 当日所有记录碳水之和（g） |

#### 2.2.3 展示内容

- **顶部数据卡**：显示总热量、蛋白质、脂肪、碳水四项数值
- **热量目标进度**：环形进度条，显示"已摄入 / 目标热量"
- **餐次分布**：按早餐、午餐、晚餐、加餐分别展示热量占比（饼图或条形图）
- **记录列表**：当日详细记录列表（复用模块一的列表展示）

#### 2.2.4 日期导航

- 支持左右切换日期（前一天/后一天）
- 支持点击日期弹出日历选择器，快速跳转到指定日期
- 有记录的日期在日历上标注小圆点

---

### 模块三：趋势统计

#### 2.3.1 功能说明

以周和月为维度展示饮食数据趋势，帮助用户了解长期饮食习惯。

#### 2.3.2 周视图

- 展示最近 7 天的热量变化折线图
- 标注每日热量目标线作为参考
- 显示周平均热量、最高/最低日热量
- 支持切换查看蛋白质/脂肪/碳水的周趋势

#### 2.3.3 月视图

- 展示最近 30 天的热量变化折线图
- 显示月平均热量、达标天数（在目标±10%范围内）
- 支持切换查看蛋白质/脂肪/碳水的月趋势
- 月度营养素摄入比例饼图（蛋白质/脂肪/碳水占总热量百分比）

#### 2.3.4 日历视图

- 月历形式展示，每个日期格显示当日总热量
- 超标日期（超过目标热量）用红色标注
- 达标日期用绿色标注
- 无记录日期灰色显示

---

### 模块四：个人目标设置

#### 2.4.1 功能说明

用户设置自己的每日饮食目标，系统根据目标展示进度和建议。

#### 2.4.2 可设置项

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 每日热量目标 | 单位 kcal | 2000 kcal |
| 蛋白质目标占比 | 占总热量的百分比 | 20% |
| 脂肪目标占比 | 占总热量的百分比 | 30% |
| 碳水目标占比 | 占总热量的百分比 | 50% |

> 三个占比之和固定为 100%，调整一个时自动重新分配其余两个。

#### 2.4.3 交互细节

- 目标设置页面独立，从侧边栏或设置入口进入
- 修改目标后立即生效，影响所有统计展示
- 提供常见目标的快捷模板（减脂/增肌/维持体重）

### 模块五：AI 功能（第二期）

#### 2.5.1 自然语言录入

**功能描述**：用户通过自然语言描述饮食内容，AI 自动解析为结构化饮食记录。

**使用场景**：
- "中午吃了一碗米饭、一份宫保鸡丁、一杯橙汁"
- "早餐吃了两个鸡蛋和一片全麦面包"
- "刚才吃了一袋薯片大概50克"

**处理流程**：
1. 用户输入自然语言描述
2. AI 解析提取食物列表和用量
3. 匹配食物库，计算热量和营养素
4. 展示解析结果供用户确认或修改
5. 用户确认后生成饮食记录

#### 2.5.2 饮食建议

**功能描述**：根据用户的目标和近期饮食数据，AI 给出个性化饮食建议。

**建议类型**：
- 今日剩余热量建议：根据已摄入和目标差距，推荐晚餐吃什么
- 营养结构调整建议：指出蛋白质/脂肪/碳水的偏差并给出调整方向
- 替代食物推荐：推荐更健康或更低热量的替代选择

#### 2.5.3 智能报告

**功能描述**：AI 生成周/月饮食分析报告。

**报告内容**：
- 饮食习惯总结（高频食物、偏好分析）
- 营养均衡评分
- 改善建议与行动计划

## 三、页面结构

### 3.1 页面列表

| 页面 | 路径 | 说明 |
|------|------|------|
| 饮食记录 | /diet/record | 记录列表 + 新增/编辑入口 |
| 每日统计 | /diet/daily | 当日统计总览 |
| 趋势统计 | /diet/trend | 周/月趋势图表 |
| 目标设置 | /diet/goal | 个人目标配置 |
| AI 助手 | /diet/ai | AI 录入和建议（二期） |

### 3.2 导航结构

在现有侧边栏新增"饮食管理"一级菜单，展开后包含上述子页面。每个页面顶部保持一致的日期选择器。

## 四、数据模型

### 4.1 饮食记录表 (diet_record)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | BIGINT | 用户ID |
| record_date | DATE | 记录日期 |
| meal_type | TINYINT | 餐次（1早餐 2午餐 3晚餐 4加餐） |
| food_name | VARCHAR(100) | 食物名称 |
| amount | DECIMAL(8,2) | 食用量 |
| unit | VARCHAR(10) | 单位（g/份/ml） |
| calories | DECIMAL(8,2) | 热量 kcal |
| protein | DECIMAL(8,2) | 蛋白质 g（可空） |
| fat | DECIMAL(8,2) | 脂肪 g（可空） |
| carbs | DECIMAL(8,2) | 碳水 g（可空） |
| remark | VARCHAR(255) | 备注（可空） |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| is_deleted | TINYINT | 逻辑删除 |

### 4.2 用户饮食目标表 (diet_goal)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | BIGINT | 用户ID（唯一） |
| daily_calories | INT | 每日热量目标 kcal |
| protein_ratio | DECIMAL(4,2) | 蛋白质目标占比 % |
| fat_ratio | DECIMAL(4,2) | 脂肪目标占比 % |
| carbs_ratio | DECIMAL(4,2) | 碳水目标占比 % |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

## 五、接口设计

### 5.1 饮食记录接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/diet/record | 新增饮食记录 |
| PUT | /api/diet/record/{id} | 修改饮食记录 |
| DELETE | /api/diet/record/{id} | 删除饮食记录 |
| GET | /api/diet/record/list | 查询指定日期的记录列表 |
| GET | /api/diet/record/daily-summary | 查询指定日期的营养汇总 |

### 5.2 目标设置接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/diet/goal | 获取当前用户目标 |
| PUT | /api/diet/goal | 设置/更新目标 |

### 5.3 趋势统计接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/diet/trend/weekly | 查询指定周的每日热量汇总 |
| GET | /api/diet/trend/monthly | 查询指定月的每日热量汇总 |

### 5.4 AI 接口（二期）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/diet/ai/parse | 自然语言解析为饮食记录 |
| GET | /api/diet/ai/suggestion | 获取饮食建议 |
| GET | /api/diet/ai/report | 生成饮食分析报告 |
